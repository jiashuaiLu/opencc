# OpenCC 快速启动指南

## 🚀 5分钟快速上手

### 第一步：安装依赖（约2分钟）

```bash
cd opencc
npm install
```

### 第二步：启动应用（约10秒）

```bash
npm run electron:dev
```

### 第三步：配置代理（约1分钟）

1. 应用启动后，进入"配置管理"页面
2. 填写配置信息：
   - **配置名称**: 例如 "OpenAI-Production"
   - **API Key**: 你的 API 密钥（如：`sk-xxxxx`）
   - **Base URL**: 选择或输入 API 端点
   - **代理端口**: 默认 8787
3. 点击"保存配置"

### 第四步：启动服务（约5秒）

1. 回到"仪表盘"页面
2. 点击"启动服务"按钮
3. 等待服务启动完成

### 第五步：配置 Claude Code（约30秒）

应用会自动配置 `~/.claude/settings.json` 文件，你也可以手动配置：

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "你的API密钥",
    "ANTHROPIC_BASE_URL": "http://localhost:8787"
  }
}
```

### 第六步：开始使用！

现在你可以使用 Claude Code 了：

```bash
claude "你好，世界！"
```

## 🔧 故障排除

### 问题1：端口被占用

**症状**: 启动失败，提示端口已被占用

**解决方案**:
```bash
# 检查端口占用
lsof -i :8787

# 关闭占用进程
kill -9 <PID>

# 或修改应用端口
```

### 问题2：Claude Code 连接失败

**症状**: Claude Code 无法连接到代理服务

**解决方案**:
1. 检查代理服务是否已启动
2. 检查配置文件 `~/.claude/settings.json`
3. 测试代理连接：
```bash
curl http://localhost:8787/health
```

### 问题3：API Key 无效

**症状**: 请求失败，提示 API Key 错误

**解决方案**:
1. 检查 API Key 是否正确
2. 检查 Base URL 是否正确
3. 检查 API Key 是否有权限

### 问题4：日志不显示

**症状**: 日志页面为空

**解决方案**:
1. 检查日志级别设置
2. 点击"刷新"按钮
3. 重启应用

## 📈 性能优化建议

1. **定期清理日志**
   - 设置日志保留天数
   - 定期清理旧日志

2. **合理设置端口**
   - 使用未被占用的端口
   - 避免使用系统保留端口

3. **优化配置**
   - 使用稳定的 API 端点
   - 合理设置超时时间

1. **保护 API Key**

2. **本地使用**
   - 仅在本地使用代理服务

3. **数据备份**

   - 导出重要数据
